# Bridge Command Interface

This bridge exposes a JSON-RPC-style transport between the web runtime and the Rust shell.
An AI agent can rely on this document alone to inspect project files, load a timeline JSON,
edit clips and tracks, and save the result back through `timeline.save`.

## Transport Envelope

### Request Envelope Schema

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["id", "method", "params"],
  "additionalProperties": false,
  "properties": {
    "id": {
      "type": "string",
      "minLength": 1,
      "description": "Opaque request identifier generated by the caller."
    },
    "method": {
      "type": "string",
      "minLength": 1
    },
    "params": {
      "type": ["object", "array", "string", "number", "integer", "boolean", "null"]
    }
  }
}
```

### Response Envelope Schema

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["id", "ok", "result", "error"],
  "additionalProperties": false,
  "properties": {
    "id": {
      "type": "string",
      "minLength": 1
    },
    "ok": {
      "type": "boolean"
    },
    "result": {
      "description": "Method-specific payload. Null on failure."
    },
    "error": {
      "type": ["string", "null"],
      "description": "Null on success. Descriptive error string on failure."
    }
  }
}
```

## Path Safety Rules

- Any path containing `..` is rejected.
- Absolute paths are rejected unless they are inside the current user's home directory or `std::env::temp_dir()`.
- Relative paths are resolved relative to the shell process working directory.
- Existing read/list/load targets are canonicalized before access, so symlinks cannot escape the allowed roots.
- Write/save requests validate the canonicalized parent directory and reject symlink final targets that resolve outside the allowed roots.
- All filesystem and timeline methods fail with `ok: false` and an error string when the sandbox blocks access.

## Timeline Data Model

`timeline.load` returns the stored timeline JSON directly as `response.result`.
`timeline.save` expects the same shape inside `params.json`.

```json
    "metadata": {
      "type": "object",

      "additionalProperties": false,
      "properties": {
        "name": {
          "type": "string",
          "minLength": 1
        },
        "fps": {
          "type": "number",
          "exclusiveMinimum": 0
        },
        "width": {
          "type": "integer",
          "minimum": 1
        },
        "height": {
          "type": "integer",
          "minimum": 1
        },
        "durationMs": {
          "type": "integer",

        },
        "background": {
          "type": "string",
          "description": "Optional canvas background, typically a CSS color or gradient token."
        }
      }
    },
    "tracks": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["id", "name", "kind", "enabled", "locked", "clips"],
        "additionalProperties": false,
        "properties": {
          "id": {
            "type": "string",
            "minLength": 1
          },
          "name": {
            "type": "string",
            "minLength": 1
          },
          "kind": {
            "type": "string",
            "enum": ["video", "audio", "title", "effect"]
          },
          "enabled": {
            "type": "boolean"
          },
          "locked": {
            "type": "boolean"
          },
          "clips": {
            "type": "array",
            "items": {
              "type": "object",
              "required": [
                "id",
                "sceneId",
                "label",
                "startMs",
                "durationMs",

                "layer",
                "assetPath",

              ],
              "additionalProperties": false,
              "properties": {
                "id": {
                  "type": "string",
                  "minLength": 1
                },
                "sceneId": {
                  "type": "string",
                  "minLength": 1,
                  "description": "Scene block id. Prefer ids returned by scene.list for generated visual clips."
                },
                "label": {
                  "type": "string",
                  "minLength": 1
                },
                "startMs": {
                  "type": "integer",
                  "minimum": 0,
                  "description": "Clip start position on the timeline in milliseconds."
                },
                "durationMs": {
                  "type": "integer",
                  "minimum": 1,
                  "description": "Rendered clip duration in milliseconds."

                  "type": "integer",

```

### Timeline Editing Rules

- Preserve `version` unless the app explicitly upgrades the timeline format.
- Preserve unknown `props` keys when editing a clip. Only change the keys you intend to modify.
- Keep `track.id` and `clip.id` stable when editing existing items. Generate new unique ids when adding tracks or clips.
- Keep `durationMs >= 1`, `startMs >= 0`, and `offsetMs >= 0`.
- Keep `assetPath` `null` for generated clips and a string path for file-backed clips.
- `sceneId` should correspond to a valid scene block. This round returns stub ids: `gradientBg`, `text`, `shape`, `image`, `counter`.
- Sorting clips by `startMs` within each track is recommended before saving.

## Commands

### `fs.read`

- `method`: `"fs.read"`
- `params schema`:

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["path"],
  "additionalProperties": false,
  "properties": {
    "path": {
      "type": "string",
      "minLength": 1
    }
  }
}
```

- `result schema`:

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["path", "contents"],
  "additionalProperties": false,
  "properties": {
    "path": {
      "type": "string"
    },
    "contents": {
      "type": "string",
      "description": "UTF-8 file contents."
    }
  }
}
```

- `example`:

```json
{
  "id": "read-1",
  "method": "fs.read",
  "params": {
    "path": "./projects/demo/timeline.json"
  }
}
```
