NextFrame · 技术方案

视频 = 纯函数

frame-pure · AI-native · CLI-first

所有视频编辑器都在围绕一件事：给定一个时间线和当前时刻，吐出这一帧的像素。

我们把这件事写成一个字面意义的纯函数：render(timeline, t) → pixels。

同样的 JSON + 同样的 t，永远产同样的像素。没有累积状态，没有随机数，没有时间依赖。

one-liner ((timeline.json, t) → 像素)。纯函数。AI 写 JSON，人类调时长，导出 MP4。 
function renderAt(timeline: Timeline, t: number): Canvas
唯一的底层合约。整个系统都从这一行推出来。

L1 · Scene 库

21 个 scene，每个都是 draw 函数

一个 scene 就是一个 JS 函数：scene(t, params, ctx, globalT) → 往 Canvas2D 上画。

没有状态，没有实例，没有生命周期。scene 是无状态的"画笔"，engine 是调度者。

  const w = ctx.canvas.width, h = ctx.canvas.height;
  const { hueA=270, hueB=200, intensity=1 } = params;
  const g = ctx.createLinearGradient(0, 0, w, h);
  g.addColorStop(0, `hsla(${hueA + Math.sin(t*0.3)*20}, 70%, 20%, ${intensity})`);
  g.addColorStop(1, `hsla(${hueB}, 70%, 10%, ${intensity})`);
  ctx.fillStyle = g;
  ctx.fillRect(0, 0, w, h);
  // ... film grain using hash(x,y,t) ...
}

符号时间

AI 不写秒数，只写关系

直接写 start: 5.0 对 AI 是地狱——改任何一个 clip 时长，所有下游秒数全崩。

我们让 clip 的 start 可以是一个表达式：{after: 'clip-drum-1', gap: 0.5} 或 {at: 'marker-drop'}。

engine 启动时做拓扑排序展开成秒，环检测，越界报错带 hint。AI 只写关系，永远对。

input
{ start: { after: 'clip-intro',
           gap: 0.3 },
  dur: 4 }
resolved
{ start: 4.8, dur: 4 }
// clip-intro ends at 4.5
0.1s 网格量化，避免浮点污染
多轨道合成

每条 track 叠加成最终帧

21 个 scene 都是为"独占全屏"写的，会画一个大背景矩形。直接叠加的话上层永远盖掉下层。

engine 的 renderAt 给 track 0 直接画；track 1+ 每个 clip 先画到一张独立的 offscreen canvas，再用 globalCompositeOperation = 'lighten' 合成到主画布。

lighten = max(bg, src)。scene 的黑背景被 lighten 掉不影响下层，scene 的亮像素（文字/图形）透出来覆盖下层。

效果：真正的多层叠加，不改任何 scene 代码。

主画布 ←──── track 0 (fluidBackground, source-over, 占满) ←── track 1 (meshGrid, offscreen + lighten, 只亮像素透出) ←── track 2 (kineticHeadline, offscreen + lighten, 只字透出) ←── track 3 (shapeBurst, offscreen + lighten, 只粒子透出) ←── track 4 (cornerBadge, offscreen + lighten, 只角标透出) ↓ 最终合成帧 (6 层可见)
per-clip blend: 'source-over' | 'screen' | 'multiply' | 'difference' 可覆盖默认
6 道防呆

AI 改 JSON 之前必须过 6 道 gate

gate检查什么失败时
1. Schemaschema 字段 / duration > 0 / tracks 非空 / id 唯一BAD_SCHEMA / BAD_DURATION / NO_TRACKS / DUP_CLIP_ID
2. 符号时间所有 {at,after,before,sync} 引用都能解析TIME_REF_NOT_FOUND (带 available 列表)
3. 资产存在assets[].path 文件真的存在于磁盘MISSING_ASSET (warning)
4. Scene 引用clip.scene 在 REGISTRY 里UNKNOWN_SCENE (带 8 个可用 scene 建议)
5. 时间范围clip [start, start+dur] ⊂ [0, duration]TIME_OUT_OF_RANGE
6. 轨内重叠同一 track 上 clip 是否重叠CLIP_OVERLAP (warning)
每个 gate 返回 structured error，AI 可以直接读 hint 自己修。不 throw，不 crash。
L1 · Render targets

scene 函数 → 像素的路径

napi-canvas （当前默认）

Node 原生模块（Rust/skia）。~300ms/帧@1080p。CPU。支持系统字体注册（macOS 装 STHeiti/Hiragino）。MP4 导出走这条路。

POC-D · 单帧 377ms · 12s 视频 80s

浏览器 Canvas2D （规划中）

同一份 scene JS 代码，直接在浏览器里跑。WebKit 的 CoreText 白嫖全部 CJK 字体。预览瞬时，scrub 无延迟。

POC-A wry · 全字体覆盖

puppeteer headless （备选）

启动 headless Chrome 渲染，再截帧写 ffmpeg。字体完美但最慢。用在需要 CJK 的 MP4 导出。

POC-C · 883ms/帧

wgpu GPU （未来）

Rust + wgpu shader 渲染。14× 加速。scene 需要从 Canvas2D 改成 shader，工程量大。

POC-Q · 3.69ms/帧

rust-skia （备选）

同 napi-canvas 底层，但不走 Node，直接 Rust 二进制。

POC-E · 51ms/帧

wasm （玩具）

Rust 编译成 wasm 在浏览器跑。不实用，但证明可移植。

POC-S · 49ms/帧

AI 循环

验证过两次：两个独立 sonnet subagent 分别用 13 次和 18 次工具调用，自主做出 13s / 14s 完整视频。零干预。

为什么这样

现有工具给人用，NextFrame 给 AI 用

剪映、Premiere、DaVinci 都围绕人类交互设计——时间轴 UI 是主要入口，JSON 是副产品。

NextFrame 反过来：JSON 是源头，UI 只是 JSON 的一种可视化。AI 直接写 JSON，CLI 直接消费 JSON，HTML 只是给人看的旁观者。

这让 AI 从"一个额外的按钮"变成"和人类平等的用户"。用户的 AI Director 面板就是这个理念的实证：输入一句话，sonnet 通过 CLI 改 timeline，渲染出 MP4。整条链路没有任何地方是"专门给 AI 的"——AI 用的就是人类用的同一个 CLI。

frame-pure 是前提，contract-first 是防漂，CLI-first 是接口。三件事凑起来 = NextFrame。
