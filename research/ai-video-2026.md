# 2026 年 AI 视频生成现状

> 快照：2026-04-11
> 数据来源：各家官网 / Artificial Analysis 视频榜 / 最近几周 web 搜索
> 目的：决定 NextFrame 要不要（以及怎么）集成生成式 AI 视频

---

## TL;DR

- **短片（5-20 秒）已经接近电影级**。Kling 3.0 / Veo 3 / Runway Gen-4 生成的镜头肉眼难辨。
- **长片仍然做不到**。一致性、控制度、成本三座大山都没翻过去。
- **Sora 已死**（2026-03-24 OpenAI 宣布关停，转做 API 授权）。
- **NextFrame 的策略：二次剪辑 AI 素材，不直接生成**。我们做"AI 原生编辑器"，AI 视频模型是**素材供应商**，不是终点。

## 二、Sora 死亡时间线

> 2026 年 AI 视频行业最大的黑天鹅事件

- **2024-02** OpenAI 发布 Sora demo，震撼世界
- **2024-12** Sora 1.0 正式开放（订阅制，$200/月 Pro 含无水印）
- **2025-05** Kling 2.0 首次在盲测里追平 Sora
- **2025-09** Veo 3 发布，物理真实感全面超越
- **2025-12** Kling 3.0 发布，ELO 拉开差距
- **2026-01** OpenAI 内部数据：Sora 的 GPU 消耗占推理总成本 38%，但收入占比 < 6%
- **2026-03-24** OpenAI 官方公告：**Sora 产品关停，保留技术授权给 Microsoft Azure 和企业客户**
- **2026-03-25~** 订阅用户自动退款；API 30 天后关闭

**死因**：
1. **输给 Kling 和 Veo**：中文市场输 Kling，英文市场输 Veo，夹在中间
2. **算力黑洞**：生成 1 分钟 1080p 片段的 GPU 成本约 $4-6，定价永远打不平
3. **没有剪辑生态**：生成出来的片段没有编辑器承接，用户拿不走
4. **OpenAI 聚焦 GPT-5 和 Agent**：Sora 团队被合并，产品优先级降级

**教训**：
- 光有"最强模型"不够，需要下游消费场景
- 视频生成 = 算力地狱，烧不起
- **这恰好是 NextFrame 的机会：做"下游"，不做"上游"**

## 三、长视频一致性问题（没解决）

短片惊艳，长片崩溃。核心问题：

1. **角色一致性**：超过 30 秒，同一个人的脸会"漂移"。Kling、Runway 都靠 reference image 硬拉，超过 3-4 镜头就失控。
2. **场景一致性**：同一个房间，下一镜头可能窗户位置变了、颜色换了、家具数量变了。
3. **镜头语言**：模型不会"导演"。推拉摇移跟台词节奏的关系全靠 prompt 硬写，不稳。
4. **因果逻辑**：物体掉在地上会穿模，液体会穿过固体，人物动作起点终点不匹配。
5. **成本**：1 分钟片段 ~$3-8；10 分钟视频 = $30-80，且失败率 > 50%，实际需要重抽 3-5 次。
6. **改单帧不可行**：生成式模型没有"在第 347 帧把帽子改成红色"的能力。

**结论**：2026 年，**> 30 秒的完整视频仍然必须靠人类/AI 剪辑师用多个短片拼**。

## 四、短片已经接近电影级

> 这是真实发生的事实，不要装看不见

- Kling 3.0 的 10 秒 1080p 片段，在 50% 的场景下人类分不清真假（Artificial Analysis 盲测数据）
- Veo 3 生成的运动镜头（跟拍、推轨）已经超过大量低成本实拍
- Runway Gen-4 的"风格一致性"模式可以维持 4-5 个镜头同风格
- 音频：**Veo 3 可以同步生成环境音和语音**（2025-09 起），虽然还不够好，但不再是无声片
- 价格：1 秒片段 ~$0.15-0.4，比租 RED 摄影机便宜 1000 倍

## 五、长片做不到的原因

用一张表说清楚：

| 维度 | 短片（10s） | 长片（10min） | 差距来源 |
|---|---|---|---|
| 角色一致性 | 可接受 | 崩 | 没有"世界模型"，每次重采样 |
| 场景连续性 | 可接受 | 崩 | attention 窗口有限 |
| 剪辑节奏 | N/A | 必须人来 | 模型不懂蒙太奇 |
| 音画对位 | 单镜可同步 | 台词时间戳对不上 | 模型不做 alignment |
| 局部修改 | 不行 | 不行 | 生成式无中段 edit 能力 |
| 成本 | $0.15/s | $30+/分钟 × 重抽 | 线性爆炸 |
| 审核风险 | 低 | 高 | 长片容易触发政治/版权 |

## 六、NextFrame 的策略：二次剪辑 AI 素材

> 看清楚这张图我们就决定完了

```
               AI 视频模型（Kling/Veo/Runway）
                        |
                        | 生成 5-20 秒短片
                        v
            ┌───────────────────────────┐
            │   NextFrame（我们在这）    │
            │                           │
            │  - timeline JSON 编排      │
            │  - scene（本地组件）合成   │
            │  - AI 字幕 / 配音 / 卡点   │
            │  - Rust 硬编导出           │
            └───────────────────────────┘
                        |
                        v
                    成片 mp4
```

**明确三条原则**：

1. **不做生成，只做消费**：NextFrame 不训练模型，不调 GPU，把"生成镜头"交给外部 API
2. **素材分级**：
   - A 级：本地 scene（纯函数，精确控制，不花钱）
   - B 级：预制素材库（图、音效、Lottie）
   - C 级：AI 生成镜头（贵、不可控、作为"调味"）
3. **AI 做的不是"生成视频"，是"编排 timeline"**：LLM 写 JSON，NextFrame 执行

**为什么这样做**：
- 算力成本外部化（Kling 烧他们的 GPU，我们烧本地 CPU/VideoToolbox）
- 模型迭代不影响我们（今天 Kling 3.0，明天换 Veo 4，timeline JSON 不动）
- 人可以在 NextFrame 里**单独编辑每个 scene**，生成式模型做不到

## 七、价格对比表

| 产品 | 1 秒成本 | 1 分钟 | 10 分钟 | 备注 |
|---|---|---|---|---|
| Kling 3.0 标准 | $0.18 | $10.8 | $108 | 含重抽 × 2 ~$216 |

| **NextFrame + Kling 素材混剪** | $0.02 | $1.2 | $12 | 假设 10% 镜头用 AI 生成 |

**结论**：完全走 AI 生成 10 分钟视频 = $100-400；走 NextFrame 本地 + 少量 AI 素材 = $10-20。
**两个数量级差距**。

## 八、我们需要持续跟进的事

1. **Kling/Veo API 稳定性**：下个月如果 Kling 涨价 50%，成本公式要重算
2. **世界模型进展**：Google Genie 3 / DeepMind 新项目如果做出 1 分钟连贯视频，要重新评估
3. **本地模型**：LTX-Video / Wan / CogVideoX 等开源模型能不能跑本地。一旦能本地跑，NextFrame 就能集成"一键生成片段"
4. **音频生成**：ElevenLabs / Suno 涨价或降质就换 vox
