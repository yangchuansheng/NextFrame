# NextFrame v0.2 Clip Architecture

## Core Concept

A clip is the atomic unit on the timeline. It combines six orthogonal dimensions:

```
clip = scene + keyframes + effects + filters + transition + audio
       content  animation   in/out    post      between     sound
```

## Six Dimensions

### 1. Scene (v0.1 done)

What to draw. Pure function: `(t, params, ctx) → pixels`

```
src/scenes/
  index.js          registry + META
  auroraGradient.js
  textOverlay.js
  ...33 total
```

No changes needed. Scenes stay content-only.

### 2. Keyframe (v0.2 new)

Any param value can be a keyframe array instead of a static value.

```json
"params": {
  "text": "HELLO",
  "fontSize": { "keys": [[0, 24], [1, 48], [4, 48], [5, 24]], "ease": "easeInOut" },
  "color": { "keys": [[0, "#ff0000"], [2.5, "#0000ff"]], "ease": "linear" }
}
```

Engine resolves keyframes before passing to scene:
```
render.js:
  const resolvedParams = resolveKeyframes(clip.params, localT);
  entry.render(localT, resolvedParams, ctx, t);
```

New file: `src/engine/keyframes.js`
- `resolveKeyframes(params, t)` — walks params, interpolates keyed values
- Number interpolation: lerp + easing
- Color interpolation: HSL lerp
- String: snap to nearest keyframe (no interpolation)
- Easing: linear, easeIn, easeOut, easeInOut, spring

### 3. Effect (v0.2 new)

Enter/exit animations applied after scene renders.

```
src/effects/
  index.js       registry + META
  fadeIn.js       opacity 0→1
  fadeOut.js      opacity 1→0
  slideUp.js      translateY + opacity
  slideDown.js
  scaleIn.js      scale 0→1 + opacity
  scaleOut.js
  blur.js         blur 10→0
  typewriter.js   clip-path reveal
```

Each effect: `(ctx, canvas, progress, params) → ctx transform`

```js
// fadeIn.js
export const META = { id: "fadeIn", params: [{ name: "dur", type: "number", default: 0.5 }] };
export function apply(ctx, canvas, progress) {
  ctx.globalAlpha = progress;  // 0→1
}
```

Engine applies in render.js:
```
scene.render(localT, params, offCtx);
applyEnterEffect(offCtx, offCanvas, localT, clip.effects.enter);
applyExitEffect(offCtx, offCanvas, localT, clip.dur, clip.effects.exit);
ctx.drawImage(offCanvas, 0, 0);
```

### 4. Filter (v0.2 new)

Post-processing on rendered clip or entire track.

```
src/filters/
  index.js        registry + META
  warmTone.js     warm color grading
  coolTone.js     cool/blue shift
  grayscale.js    desaturate
  sepia.js        vintage brown
  filmGrain.js    noise overlay
  blur.js         gaussian blur
  sharpen.js      unsharp mask
  vignette.js     edge darkening (migrate from scene)
```

Each filter: `(imageData, params, t) → imageData`

Works on pixel data (getImageData/putImageData).

```json
"filters": [
  { "type": "warmTone", "intensity": 0.6 },
  { "type": "filmGrain", "amount": 0.03 }
]
```

### 5. Transition (v0.2 new)

Cross-fade between two overlapping clips.

```
src/transitions/
  index.js        registry + META
  dissolve.js     opacity cross-fade
  wipe.js         directional wipe (left/right/up/down)
  zoom.js         zoom into next clip
  slide.js        slide old out, new in
  blur.js         blur out old, blur in new
```

Each transition: `(ctxA, canvasA, ctxB, canvasB, progress, params) → merged canvas`

Applied when two clips overlap on same track, or via explicit transition property.

### 6. Audio (v0.1 partial, v0.2 expand)

Already: `--audio=narration.mp3` mux into final mp4.
Add: per-clip audio, sound effects library.

```json
{
  "tracks": [
    { "id": "a1", "kind": "audio", "clips": [
      { "id": "bgm", "start": 0, "dur": 60, "src": "music.mp3", "volume": 0.3 },
      { "id": "sfx-1", "start": 5, "dur": 1, "effect": "whoosh" }
    ]}
  ]
}
```

Lower priority than keyframes/effects/filters.

## Render Pipeline (v0.2)

```
for each clip active at time t:
  1. resolveKeyframes(clip.params, localT)     → resolved params
  2. scene.render(localT, resolved, offCtx)    → content pixels
  3. applyEffects(offCtx, localT, clip)        → enter/exit animation
  4. applyFilters(offCanvas, clip.filters)      → post-processing
  5. applyTransition(prev, current, progress)   → cross-fade
  6. composite to main canvas (blend mode)      → final frame
```

## File Structure (v0.2)

```
src/
  engine/
    render.js        ← add steps 3-5
    keyframes.js     ← NEW: interpolation engine
    validate.js      ← add effect/filter/transition validation
  scenes/            (unchanged, 33 scenes)
  effects/           ← NEW
    index.js
    fadeIn.js, slideUp.js, scaleIn.js, ...
  filters/           ← NEW
    index.js
    warmTone.js, grayscale.js, filmGrain.js, ...
  transitions/       ← NEW
    index.js
    dissolve.js, wipe.js, zoom.js, ...
```

## Implementation Order

1. **keyframes.js** — interpolation engine (unlocks everything)
2. **effects/** — enter/exit (most visible impact)
3. **filters/** — post-processing (easy wins)
4. **transitions/** — cross-fades (needs overlap detection)
5. **audio expansion** — sound effects library

## Backward Compatibility

- v0.1 clips work as-is (no effects/filters/keyframes = skip those steps)
- `schema: "nextframe/v0.1"` still valid
- New features require `schema: "nextframe/v0.2"`
- Gallery previews show all dimensions independently
