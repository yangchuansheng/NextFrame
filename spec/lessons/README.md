# Lessons — Personal Knowledge Base

Accumulated experience from building NextFrame. Written for the human, not for AI.

## Structure

```
lessons/
├── README.md           ← this file
├── pitfalls/           ← things that went wrong and why
├── decisions/          ← reasoning behind key choices
├── techniques/         ← reusable patterns and approaches
└── reflections/        ← bigger picture thinking
```

## Format

Each lesson is one markdown file, named by topic:

```markdown
# {Topic}

## Context
What was happening when I learned this.

## Lesson
What I learned.

## Evidence
What happened that proved it.
```

## Rules

- Write when fresh — don't wait
- One topic per file
- Be honest — include mistakes
- This is private — write for yourself, not an audience
