NextFrame

AI-Native Video Editor — Complete System Architecture

f(timeline, t) → frame

产品定位

NextFrame 是 AI 原生的视频编辑器。AI 写 JSON 描述视频内容，系统自动渲染成 MP4。

核心理念
f(timeline, t) → frame
任意一帧 = 纯函数，给定时间线和时间点，确定性输出像素
AI 驱动
JSON → MP4
AI 只需要写 timeline JSON，不需要操作 GUI
帧纯函数
Frame-Pure
每一帧独立可计算，支持并行、跳帧、随机访问

整体数据流

🤖

1. AI 写内容

AI 生成 Timeline JSON 描述多轨道视频内容 每个 clip 指定 scene + 参数 + 时间

📦

harness-gen.js 把 timeline + 所有 scene 代码 + engine + effects + filters 打包成一个自包含 HTML

HTML 直接在浏览器打开 拖动时间轴实时预览 所见即所得

🎬

WKWebView 加载 HTML CALayer 逐帧截图 VideoToolbox H.264 硬编 → MP4 文件

Timeline JSON 结构

Timeline
project width: 3840, height: 2160, fps: 30
background #05050c 全局底色
assets[] 素材库：image / audio / video / subtitle / font
chapters[] 章节标记（时间范围）
markers[] 时间点标记（锚点）
tracks[] 多轨道，从底到顶渲染
kind video | audio
muted / locked / solo 轨道控制
clips[] 每个 clip 是一段内容
scene 场景 ID：auroraGradient / kineticHeadline / ...
start / dur 起始时间 + 持续时间（支持 SymbolicTime）
params 场景参数（支持 Keyframes 动画）
blend 混合模式：screen / lighten / multiply ...
effects 进出场：fadeIn / slideUp / scaleIn ...
filters 后处理：grayscale / sepia / filmGrain ...

混合渲染架构

WKWebView 是渲染引擎，HTML 页面是舞台

Canvas 2D流体渐变 · 粒子系统 · 手绘特效 · 数据可视化逐像素控制的动画效果
DOM + CSS文字排版 · 富文本 · CSS 动画 · Flex/Grid 布局系统字体渲染清晰 · 自动换行 · 布局引擎
SVG图表 · 图标 · 矢量图形 · 路径动画无损缩放 · 4K 自动清晰
WebGL / WebGPU3D 场景 · 着色器 · GPU 粒子GPU 直接计算 · 复杂视觉
<video> / <img>视频素材 · 图片素材原生解码 · 零额外代码
CALayer.renderInContext() 截图 = 截整个页面最终像素，不管用什么技术画的

核心概念全景

概念数量说明举例
Scenes 场景39画面原子单元，每个是一个纯函数auroraGradient / kineticHeadline / barChart / cornerBadge
Effects 特效6clip 进出场动画fadeIn / fadeOut / slideUp / slideDown / scaleIn / scaleOut
Filters 滤镜5色彩后处理grayscale / sepia / warmTone / coolTone / filmGrain
Blend 混合8图层合成模式source-over / lighten / screen / multiply / overlay
Transitions 转场4片段间过渡（v0.2 接入）dissolve / wipeLeft / wipeUp / zoomIn
Keyframes 关键帧∞任何参数可随时间动画{ keys: [[0,0],[1,100]], ease: 'easeOut' }
SymbolicTime7 种相对时间表达式{ after: 'clip-1', gap: 0.5 } / { at: 'marker-1' }
Assets 素材5 类项目素材库image / audio / video / subtitle / font
Audio 音频—Web Audio 混音器volume / gainAutomation / clipStart / clipDur
Chapters 章节—带名字的时间范围{ name: 'Intro', start: 0, end: 10 }
Markers 标记—时间点锚{ name: 'drop', t: 5.0 }

一帧的渲染管线

10 步完成一帧

1
SymbolicTime 解析
把 { after: 'clip-1' } 解析为具体秒数
2
遍历 video tracks
从底层 track 到顶层 track
3
找激活的 clip
当前时间 t 在哪些 clip 的 [start, start+dur) 范围内
4
Keyframes 插值
把关键帧参数插值为当前值
5
Scene 渲染
scene(localT, params, offCtx) 画到离屏画布
6
Filters 后处理
grayscale / sepia / filmGrain 应用到离屏画布
7
Effects 动画
fadeIn / slideUp 修改 alpha 和 transform
8
Blend 合成
设置 globalCompositeOperation 混合模式
9
drawImage 合成
离屏画布绘制到主画布
10
重复 2-9
直到所有 track 的所有 clip 完成

为什么选 WKWebView

零成本整合

DOM/SVG/Canvas/WebGL/WebGPU/Video 全部现成，不用自己实现

自由选型

每个 scene 选最适合的技术，文字用 DOM，特效用 Canvas，图表用 SVG

系统字体

文字永远清晰，不用自己画字、量宽、算换行

CSS 动画

入场/出场/过渡动画用 CSS 写，GPU 自动加速

零权限

macOS 内置 WebKit，一台全新 Mac 零额外权限就能跑

一次截图全收

CALayer 截的是最终合成像素，所有技术共存的结果
