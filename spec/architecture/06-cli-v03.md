# 06 · CLI v0.3 — AI 唯一入口

## 目标

CLI 是 AI 操作 NextFrame 的唯一入口。AI 通过 CLI 命令完成：创建时间线、添加/移动/删除层、预览（ASCII/截图）、验证、生成 HTML、录制视频。

## v0.1 → v0.3 变化

| 维度 | v0.1 | v0.3 |
|------|------|------|
| 数据格式 | tracks[] → clips[] | layers[]（扁平） |
| 组件 | 30 个 canvas-only | 48 个（DOM/Canvas/SVG/WebGL/Media） |
| 引擎 | engine/ (canvas 合成) | engine-v2.js (HTML 合成) |
| 输出 | napi-canvas PNG → ffmpeg MP4 | bundle.cjs → 单文件 HTML |
| 预览 | WKWebView app | 浏览器打开 HTML |

## CLI 命令清单（v0.3）

### 创建/管理
```bash
nextframe new <name>                    # 创建新项目（生成 timeline.json 模板）
nextframe validate <timeline.json>      # 验证 timeline 格式 + 组件存在性
```

### 层操作（AI 核心工具）
```bash
nextframe add <timeline.json> --scene headline --start 2 --dur 5 --params '{"text":"Hello"}'
nextframe remove <timeline.json> --id title-1
nextframe move <timeline.json> --id title-1 --start 4
nextframe resize <timeline.json> --id title-1 --dur 8
nextframe set <timeline.json> --id title-1 --param text "World"
nextframe set <timeline.json> --id title-1 --enter "fadeIn 0.8s"
nextframe set <timeline.json> --id title-1 --transition "dissolve 0.5s"
nextframe list <timeline.json>          # 列出所有层
```

### 查看/预览
```bash
nextframe scenes                        # 列出全部 48 个组件 + 参数
nextframe scene <id>                    # 查看单个组件详情
nextframe describe <timeline.json> <t>  # 描述 t 秒时画面上有什么
nextframe gantt <timeline.json>         # ASCII 甘特图
nextframe ascii <timeline.json> <t>     # ASCII 截图（AI 视觉）
nextframe frame <timeline.json> <t>     # PNG 截图
```

### 生成/输出
```bash
nextframe build <timeline.json> -o out.html   # 生成可播放 HTML
nextframe render <timeline.json> -o out.mp4   # 录制 MP4（需要 recorder）
```

## 数据格式（v0.3 layers）

```json
{
  "width": 1920, "height": 1080, "fps": 30, "duration": 20,
  "background": "#05050c",
  "layers": [
    {
      "id": "bg",
      "scene": "auroraGradient",
      "start": 0,
      "dur": 20,
      "params": { "hueA": 265 },
      "enter": "fadeIn 0.8s",
      "exit": "fadeOut 0.5s",
      "transition": "dissolve 0.5s",
      "blend": "screen",
      "opacity": 0.5,
      "filter": "blur(2px)"
    }
  ]
}
```

## 组件注册表

48 个组件在 `runtime/web/src/scenes-v2/` 下，每个导出：
```js
export default {
  id: "headline",
  type: "dom",           // dom / canvas / svg / webgl / media
  name: "Kinetic Headline",
  category: "Typography",
  defaultParams: { text: "TITLE", fontSize: 72, ... },
  create(container, params) { ... },
  update(els, localT, params) { ... },
  destroy(els) { ... }
}
```

CLI 的 `nextframe scenes` 命令读取所有组件的 id/type/name/category/defaultParams，以 JSON 输出给 AI。

## 防呆设计

1. **validate 6 道门**：schema → 时间冲突检测 → 组件存在性 → 参数类型 → asset 路径 → 重复 ID
2. **describe** 输出每层的 id/scene/可见性/进度，AI 可用来理解当前画面
3. **ASCII 截图**：puppeteer 截图 → 灰度 → ASCII，AI 不用看图就能"看到"画面
4. **scenes 命令**：完整组件目录 + 参数说明，AI 可查阅后选用
5. **操作原子化**：每个 add/remove/set 是一次 JSON 读写，可独立回滚
