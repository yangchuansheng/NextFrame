NextFrame

AI-native frame-pure desktop video editor

v0.1 · spec draft · 2026-04-12

00核心命题

NextFrame 的全部决策都从这一行推导。

视频不是"时间线 + 时间状态"，而是函数：
render(timeline, t) → frame
给定 timeline 和任意 t，直接算出那一帧。无 cache、无累积状态。
这是和剪映/Premiere/Final Cut 本质上不一样的地方。它们底层是 timeline cursor + state machine，frame-pure 是函数。

这一点成立，下面所有东西就免费：录制、scrub、AI 编辑、并行渲染、撤销、测试。

04用户能做的事 · 三层

每条都是用户视角的动作。不写技术。

核心 6 件（缺一不算 NextFrame）

  1. 打开/保存 .nfproj
    timeline JSON 文档生命周期。可被 AI、CLI、GUI 读写。
  2. 浏览 scene 库 + 拖到时间线
    21 个内置 scene 可视化挑选 + 拖出 clip。
  3. 预览播放 / 拖动时间线
    看到自己的工作。frame-pure 让任意 t 都能立刻渲。
  4. 改 scene 参数 → 即时看见结果
    这是"frame-pure"对用户的真实意义：无延迟。
  5. 导出 MP4
    终点产物。video + audio 流。
  6. AI 一句话 → 完整 timeline
    "做一个 12 秒的产品发布开场" → JSON。AI-native 的真表达。

进阶 6 件（区分 NextFrame 和"剪映 clone"）

  1. 预览里直接拖动 scene 元素
    点 scene 里的某个元素 → 出选中框 → 拖 → 改参数 → 下一帧立即生效。Direct manipulation。
  2. 关键帧 / 参数动画
    任意数值参数都可以打 keyframe，scene 自然有动画。
  3. 多轨道 layer 合成
    V1 + V2 同时画，z 顺序、blend mode（screen/multiply/overlay…）。
  4. 导入用户素材
    视频、图片、音频、字幕真世界资产塞进 frame-pure 模型。
  5. AI 操作产品（不只是生成）
    AI 能 click、能 drag、能截图看结果、能改 JSON 修 bug。CLI + bridge 即可。
  6. 模板 / preset
    起手式：开 NextFrame → 选模板 → 改文字 → 出片。

未来 5 件（v0.2+）

  1. 实时协作 / 多人编辑
  2. 云渲染 / 分布式 export（K POC 已证）
  3. 插件系统（社区写 scene）
  4. 移动端 / Web 版
  5. 直播流 / OBS 替代

05技术决策（POC 已答）

19 个 POC 把所有"用什么"问题都问完了。

渲染引擎选什么？

分两条路并存：

  • 开发期 / 编辑器预览：D napi-canvas（drop-in JS scene，377ms 单帧，作者写 scene 不用 port 到任何其他语言）
  • 导出期 / 终极性能：Q wgpu（3.69ms 单帧，14× 快，scene 需要 port 到 WGSL，但只有"top scene"需要 port）
  • 跨平台单文件：S wasm（49ms，28KB，浏览器/node/CLI 都能跑）

scene 模型 — 函数 or 数据？

POC F 和 G 给了答案：frame-pure 函数留作"内置库的实现方式"，因为写起来短 2.1×。但预览里的 direct manipulation 通过 G 的 element-tree 模型表达，编辑器把函数渲染的结果当背景，覆一层可拖拽的元素树。两者并存。

AI 怎么操作？

POC J 证明：不需要 GUI 自动化。AI 直接读 SCENE_MANIFEST + 写 timeline.json + 调 nextframe render --t X 看结果。CLI + JSON 是 AI-native 的全部。

导出靠谁？

POC H 证明：自己写 ffmpeg pipe 几十行就够。不需要外部 recorder 二进制。

音频怎么同步？

POC O 证明：audio 是 master clock，video frame 从 sample index 推导。44100Hz / 30fps = 1470，整数对齐，零误差。

06边界 · 做和不做

做

  • frame-pure scene 库（持续扩）
  • JSON timeline 作为 source of truth
  • CLI 能完整使用 NextFrame
  • desktop 单文件二进制
  • macOS 优先（先打透）
  • 本地优先（不依赖云）
  • AI agent first-class user
  • video + audio + 字幕 真混音
  • Direct manipulation in preview

不做

  • 3A 游戏 CG / 电影级 VFX
  • 骨骼动画 / Live2D
  • DAW（不是音乐制作工具）
  • AI 视频生成本身（只编辑 AI 素材）
  • 移动端 / 浏览器云端版（v0.1）
  • 直播推流（v0.2）
  • 跨平台 Linux/Windows（v0.2）
  • 框架（不用 Tauri/Electron/React/Vue）
  • "剪映 clone"（不复制别人的 UX）
