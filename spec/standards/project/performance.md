# 06 — Performance Budget

## 时间预算

| 操作 | 预算 | 超了 = |
|------|------|--------|
| 编辑器帧渲染 | ≤ 16ms (60fps) | 卡顿 |
| 预览帧渲染 | ≤ 33ms (30fps) | 掉帧 |
| 录制帧捕获 | ≤ 100ms | 录制太慢 |
| IPC 往返 | ≤ 50ms | UI 响应慢 |
| 应用启动 | ≤ 2s | 体验差 |
| CLI 命令响应 | ≤ 500ms | AI 等太久 |
| TTS 单句合成 | ≤ 3s | 批量太慢 |
| validate 门禁 | ≤ 200ms | 影响编辑流程 |

## 内存预算

| 场景 | 预算 |
|------|------|
| 编辑器空载 | ≤ 100MB |
| 编辑 50 个 clip | ≤ 300MB |
| 录制中 | ≤ 500MB |
| CLI 空载 | ≤ 50MB |

## 并行策略

| 操作 | 策略 |
|------|------|
| 帧录制 | 最多 8 worker 并行 |
| TTS 批量 | 最多 4 并发请求 |
| 文件 I/O | 异步非阻塞 |
| FFmpeg 编码 | 单进程，硬件加速优先 |

## 性能日志

录制完成后输出 perf.jsonl：
```json
{"event":"frame","t":0.033,"render_ms":12,"capture_ms":45,"encode_ms":8}
{"event":"summary","total_ms":45000,"frames":1350,"avg_frame_ms":33.3}
```

超预算的帧标红。AI 可以读这个判断性能问题。
