视频转场类型

8 大类、25+ 种转场，实时 CSS/JS 演示。点击「播放」即可预览效果。

1. 硬切转场（Cut Transitions）

90% 的视频剪辑都是硬切。其他转场是为了强调。硬切才是默认选择。

硬切（Hard Cut）

场景 A 瞬间变成场景 B。没有动画，没有特效。零帧转场。

适用场景：对话正反打、快节奏剪辑、大多数情况。

场景 A采访镜头
场景 B反打角度
// 硬切（Hard Cut）：瞬间切换，无转场 sceneA.style.opacity = '0'; sceneB.style.opacity = '1'; // 就这样。没有动画。硬切本身就是"没有转场"。

跳切（Jump Cut）

同一场景，轻微的时间跳跃。主体突然移位。常用于 vlog 和蒙太奇。

适用场景：时间压缩、高能节奏、去掉口播中的停顿。

👤
同一场景，主体位移
150ms
// 跳切（Jump Cut）：同一场景，不同位置 // 短暂闪白来强化"跳跃"感 stage.style.background = '#fff'; subject.style.transform = `translate(${randomX}px, ${randomY}px)`; setTimeout(() => stage.style.background = '', 50);

匹配剪辑（Match Cut）

相似的形状或颜色连接两个场景。球变成星球，车轮变成时钟。

适用场景：主题关联、叙事、"艺术感"剪辑。

月亮
800ms
// 匹配剪辑（Match Cut）：形状 A 变形为形状 B // 保持圆形，改变颜色和标签 shape.style.transition = `all ${duration}ms cubic-bezier(0.16,1,0.3,1)`; shape.style.background = sceneB_color; shape.textContent = '星球'; // 形状的连续性让眼睛接受了场景的切换。

2. 溶解转场（Dissolve Transitions）

就是两层的透明度动画。情感转场的主力军。

交叉溶解（Cross Dissolve）

A 淡出的同时 B 淡入，两者重叠。转场过程中两个场景同时可见。

适用场景：时间流逝、情绪转换、梦境。

场景 A现在
场景 B回忆
1000ms
/* 交叉溶解（Cross Dissolve）：纯 CSS opacity */ .scene-a { transition: opacity ${duration}ms ease; opacity: 0; /* 淡出 */ } .scene-b { transition: opacity ${duration}ms ease; opacity: 1; /* 同时淡入 */ } /* 关键：两层同时做动画，产生重叠效果 */

淡入黑场（Fade to Black）

A 淡入黑色，停顿，然后 B 从黑色淡出。依次进行，不是同时。

适用场景：章节结束、戏剧停顿、"时间流逝"的感觉。

场景 A幕终
场景 B新篇章
800ms
// 淡入黑场（Fade to Black）：A → 黑 → B（依次进行） overlay.style.transition = `opacity ${half}ms ease`; overlay.style.opacity = '1'; // 淡入黑色 setTimeout(() => { sceneA.style.opacity = '0'; sceneB.style.opacity = '1'; // 底层切换 overlay.style.opacity = '0'; // 显露 B }, half);

淡入白场（Fade to White）

和淡入黑场一样，但经过白色。更明亮、更有希望感。

适用场景：闪回、梦幻场景、正面情绪转换。

场景 A之前
场景 B之后
800ms
// 淡入白场（Fade to White）：和淡入黑场完全一样，只是换了遮罩颜色 overlay.style.background = '#fff'; // 时序逻辑与淡入黑场相同。

品牌色过渡（Dip to Color）

与淡入黑场/白场相同的模式，但使用品牌色作为中间过渡。

适用场景：品牌内容、社交媒体片头、Logo 展示。

场景 A你的品牌
场景 B新片段

// 品牌色过渡（Dip to Color）：overlay.background = brandColor // 同样的依次淡入模式，任何颜色都行。

3. 擦除转场（Wipe Transitions）

clip-path 是关键 CSS 属性。通过动画裁剪区域来揭示场景 B。

水平擦除（Horizontal Wipe）

场景 B 从左到右（或从右到左）揭示，像拉开幕布一样。

适用场景：纪录片场景切换、新闻片段、前后对比。

场景 A之前
场景 B之后
800ms
/* 水平擦除（Horizontal Wipe）：clip-path inset 动画 */ .scene-b { clip-path: inset(0 100% 0 0); /* 完全裁剪 = 隐藏 */ transition: clip-path ${duration}ms cubic-bezier(0.16,1,0.3,1); } /* 触发: */ sceneB.style.clipPath = 'inset(0 0% 0 0)'; /* 完全显示 */ /* 反向: inset(0 0 0 100%) 实现从右到左 */

垂直擦除（Vertical Wipe）

从上到下揭示。同样的技术，不同的轴。

适用场景：竖屏滚动内容、社交媒体转场。

场景 A顶部
场景 B底部
800ms
/* 垂直擦除（Vertical Wipe） */ .scene-b { clip-path: inset(0 0 100% 0); } /* 隐藏 */ sceneB.style.clipPath = 'inset(0 0 0% 0)'; /* 从上到下揭示 */

光圈擦除（Iris Wipe）

从中心扩展的圆形揭示场景 B。经典电影技法（兔八哥结尾）。

适用场景：复古风格、喜剧、聚焦主体的开场/结尾。

场景 A焦点
场景 B揭示
800ms
/* 光圈擦除（Iris Wipe）：circle() clip-path */ .scene-b { clip-path: circle(0% at 50% 50%); } sceneB.style.transition = `clip-path ${duration}ms cubic-bezier(0.16,1,0.3,1)`; sceneB.style.clipPath = 'circle(75% at 50% 50%)'; /* 75% 半径覆盖整个矩形包括四角 */

时钟擦除（Clock Wipe）

旋转指针从 12 点方向扫过，像钟面一样揭示场景 B。

适用场景：与时间相关的转场、复古/经典电影。

场景 A之前
场景 B之后
1200ms
// 时钟擦除（Clock Wipe）：通过 JS requestAnimationFrame 动画 polygon // CSS clip-path polygon 无法在不同顶点数之间做动画， // 所以我们每帧计算 polygon 顶点： function clockFrame(progress) { // 从中心到 12 点方向构建 polygon，顺时针扫过 const angle = progress * 360; const points = [{ x: 50, y: 50 }, { x: 50, y: 0 }]; // 扫过每个角时添加角点 if (angle > 45) points.push({ x: 100, y: 0 }); if (angle > 135) points.push({ x: 100, y: 100 }); if (angle > 225) points.push({ x: 0, y: 100 }); if (angle > 315) points.push({ x: 0, y: 0 }); // 添加当前扫到的点 const rad = (angle - 90) * Math.PI / 180; points.push({ x: 50 + 70 * Math.cos(rad), y: 50 + 70 * Math.sin(rad) }); return `polygon(${points.map(p => p.x+'% '+p.y+'%').join(', ')})`; }

星形擦除（Star Wipe）⭐

星形从中心扩展。出了名的土。只能反讽使用（或者你是海绵宝宝）。

适用场景：喜剧、反讽复古、儿童内容。正经片子别用。

场景 A正经
场景 B哈哈
1000ms
// 星形擦除（Star Wipe）：五角星 polygon，从 0 缩放到完整 function starPoints(scale) { const pts = []; for (let i = 0; i < 10; i++) { const angle = (i * 36 - 90) * Math.PI / 180; const r = i % 2 === 0 ? 50 * scale : 20 * scale; pts.push({ x: 50 + r * Math.cos(angle), y: 50 + r * Math.sin(angle) }); } return `polygon(${pts.map(p=>p.x+'% '+p.y+'%').join(', ')})`; }

4. 推移转场（Slide / Push Transitions）

transform: translateX/Y + 定位。A 和 B 之间的时间关系决定了变体类型。

滑入（Slide）

场景 B 从某个方向滑入覆盖 A。A 在下面保持不动。

适用场景：App 式导航、面板展示、移动端 UI 转场。

场景 A当前视图
场景 B新视图
600ms
/* 滑入（Slide）：B 滑入覆盖 A（A 不动） */ .scene-b { transform: translateX(100%); /* 初始位于画面右侧外 */ transition: transform ${duration}ms cubic-bezier(0.16,1,0.3,1); } sceneB.style.transform = 'translateX(0%)'; /* 滑入画面 */ /* A 不动 — B 覆盖它 */

推动（Push）

A 被推出的同时 B 推入。两者同时向同一方向运动。

适用场景：轮播式导航、等权重的场景切换。

场景 A离场
场景 B入场
600ms
/* 推动（Push）：A 和 B 一起运动 */ .scene-a { transition: transform ${duration}ms ease; } .scene-b { transform: translateX(100%); transition: transform ${duration}ms ease; } /* 触发: */ sceneA.style.transform = 'translateX(-100%)'; /* A 向左退出 */ sceneB.style.transform = 'translateX(0%)'; /* B 从右进入 */

覆盖（Cover）

B 从某个方向滑入覆盖 A。与滑入类似但强调 B 覆盖 A。A 保持静止。

适用场景：叠加面板、"新内容到来"的感觉。

场景 A背景
场景 B覆盖中
600ms
/* 覆盖（Cover）：B 从底部滑入覆盖 A */ sceneB.style.transform = 'translateY(100%)'; /* 初始在下方 */ sceneB.style.transition = `transform ${duration}ms ease`; sceneB.style.transform = 'translateY(0%)'; /* 覆盖 A */

揭示（Reveal）

A 滑走，揭示底下的 B。B 一直都在那里。

适用场景："揭幕"内容、剥开层次、戏剧性揭示。

场景 B一直都在
场景 A滑走中
600ms
/* 揭示（Reveal）：A 滑走，B 在下面 */ /* B 位于 A 后面（更低的 z-index 或先绘制） */ sceneA.style.transition = `transform ${duration}ms ease`; sceneA.style.transform = 'translateX(-100%)'; /* A 退出，揭示 B */

5. 缩放转场（Zoom Transitions）

transform: scale() + opacity 组合。创造深度感和类似摄像机运动的效果。

放大（Zoom In）

场景 A 放大（scale up）直到充满/模糊，然后场景 B 以正常比例出现。

适用场景：聚焦细节、戏剧性强调、"进入"某个场景。

场景 A正在放大...
场景 B到达
800ms
/* 放大（Zoom In）：A 放大 + 淡出，然后显示 B */ .scene-a { transition: transform ${duration}ms ease, opacity ${half}ms ease ${half}ms; transform: scale(3); opacity: 0; } .scene-b { transition: opacity ${half}ms ease ${half}ms; opacity: 1; transform: scale(1); /* B 以正常比例出现 */ }

缩小（Zoom Out）

镜头从 A 拉远，揭示 B 作为更大的画面。

适用场景："全局"揭示、拉远展示背景、结尾。

场景 A特写
场景 B全景
800ms
/* 缩小（Zoom Out）：A 缩小，B 在后面出现 */ sceneA.style.transition = `transform ${d}ms ease, opacity ${d}ms ease`; sceneA.style.transform = 'scale(0.3)'; sceneA.style.opacity = '0'; sceneB.style.transition = `opacity ${d}ms ease`; sceneB.style.opacity = '1';

穿越缩放（Zoom Through）

放大 A 的细节，持续放大，从 B 中浮现。两个场景间连续运动。

适用场景：无缝场景衔接、旅行/旅程蒙太奇。

场景 A穿越中...
场景 B浮现
1000ms
/* 穿越缩放（Zoom Through）：A 放大超出画面，B 从小放大 */ // 阶段 1：A 放大 sceneA.style.transform = 'scale(5)'; sceneA.style.opacity = '0'; // 阶段 2：B 从小缩放到正常（在 A 结束前开始） setTimeout(() => { sceneB.style.opacity = '1'; sceneB.style.transform = 'scale(1)'; }, duration * 0.3);

6. 三维转场（3D Transitions）

使用 perspective 的 CSS 3D transforms。慎用 -- 炫酷但大多数场景下容易分散注意力。

方块旋转（Cube Rotate）

场景位于旋转方块的面上。舞台旋转展示下一个面。

适用场景：技术演示、游戏内容、"酷炫"转场。慎用。

场景 A正面
场景 B右面
800ms
/* 方块旋转（Cube Rotate）：CSS 3D */ .stage { perspective: 800px; } .cube { transform-style: preserve-3d; transition: transform ${duration}ms cubic-bezier(0.16,1,0.3,1); } .face-a { transform: translateZ(100px); } .face-b { transform: rotateY(90deg) translateZ(100px); } /* 旋转整个方块: */ cube.style.transform = 'translateZ(-100px) rotateY(-90deg)';

翻转（Flip）

翻页效果。A 沿 Y 轴旋转，揭示"背面"的 B。

适用场景：卡片揭示、前后对比、翻页隐喻。

场景 A正面
场景 B背面
800ms
/* 翻转（Flip）：backface-visibility 是关键 */ .wrapper { transform-style: preserve-3d; } .scene-a { backface-visibility: hidden; } .scene-b { backface-visibility: hidden; transform: rotateY(180deg); } /* 翻转: */ wrapper.style.transform = 'rotateY(180deg)'; /* backface-visibility: hidden 在旋转超过 90 度时隐藏每个面 */

折叠（Fold）

场景 A 像折纸一样折走，揭示后面的 B。

适用场景：折纸/纸张隐喻、创意揭示。

场景 B揭示
场景 A折叠中...
800ms
/* 折叠（Fold）：rotateY 配合 transform-origin 在一侧边缘 */ .scene-a { transform-origin: left center; transition: transform ${duration}ms ease; } sceneA.style.transform = 'rotateY(-90deg)'; /* 向左折走 */

7. 变形转场（Morphing Transitions）

最难实现但最惊艳。场景间的形状和颜色插值。

形状变形（Shape Morph）

场景 A 中的元素变形为场景 B 中的不同元素。需要匹配几何形状。

适用场景：品牌标识转场、Logo 动画、高端内容。

正方形
1000ms
/* 形状变形（Shape Morph）：动画 border-radius、尺寸、颜色 */ element.style.transition = `all ${duration}ms cubic-bezier(0.16,1,0.3,1)`; element.style.borderRadius = '50%'; // 正方形 → 圆形 element.style.background = sceneB_color; element.style.width = '120px'; element.style.height = '120px'; /* 复杂形状：使用 clip-path 或 SVG path morphing（SMIL/GSAP） */

颜色变形（Color Morph）

整个色板从场景 A 的配色转变为场景 B 的配色。背景、强调色，全部改变。

适用场景：氛围变化、日转夜、季节转换。

日间场景 暖色调
1000ms
/* 颜色变形（Color Morph）：颜色属性的 CSS transition */ .scene { transition: background ${duration}ms ease, color ${duration}ms ease; } scene.style.background = nightBg; scene.style.color = nightText; scene.textContent = '夜间场景'; /* CSS 自动处理颜色值之间的插值 */

8. 动态图形转场（Motion Graphics Transitions）

让 YouTube/TikTok 片头有"专业感"的东西。更复杂，通常需要 canvas 或 WebGL。

亮度/渐变擦除（Luma / Gradient Wipe）

用黑到白的渐变作为遮罩。深色区域先揭示，浅色区域后揭示。

适用场景：有机感的揭示、烟雾/云朵擦除、高端内容。

场景 A遮罩中
场景 B揭示
1000ms
/* 渐变擦除（Gradient Wipe）：动画 mask-image 位置 */ .scene-b { -webkit-mask-image: linear-gradient(to right, black 0%, transparent 50%, transparent 100%); } // 通过移动渐变断点实现动画： // 起始：black 0%, transparent 0%（不可见） // 结束：black 100%, transparent 150%（完全可见） // 这会产生柔边擦除效果。

故障（Glitch）

数字故障效果：RGB 分离、切片位移、噪点。短暂打乱画面。

适用场景：科技内容、赛博朋克美学、错误/中断主题。

场景 A稳定
场景 B故障切入

// 故障（Glitch）：快速 clip-path 切片 + 色相偏移 function glitchFrame() { const slices = 5; for (let i = 0; i < slices; i++) { const y1 = (i / slices) * 100; const y2 = ((i + 1) / slices) * 100; const offsetX = (Math.random() - 0.5) * 20; // 每个切片应用随机水平偏移 // Use clip-path: inset(y1% 0 (100-y2)% 0) on duplicated layers } // 添加 CSS filter: hue-rotate() 实现色相偏移 element.style.filter = `hue-rotate(${Math.random()*90}deg)`; } // 以 30fps 运行约 400ms，然后硬切到场景 B。

适用场景：魔法/科幻内容、灭霸响指、高端动态图形。

什么时候用什么转场？

快速决策树。拿不准就硬切。

  • 情绪转换？ → 溶解（交叉溶解或淡入黑场）
  • 同一场景，时间跳跃？ → 跳切或快速溶解
  • 新话题或新段落？ → 擦除或推动
  • 戏剧性揭示？ → 缩放（放大或缩小）
  • 活泼/高能？ → 3D 或动态图形
  • 品牌感/专业感？ → 品牌色过渡、渐变擦除
  • 科技/赛博朋克？ → 故障（Glitch）
  • 拿不准？ → 直接硬切。认真的。硬切几乎总是对的。
