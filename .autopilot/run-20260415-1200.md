## Audio Bug Root Cause Analysis

**链路**: build.js 把 timeline.audio.src (绝对路径如 `/Users/xxx/clip.mp4`) 嵌入 HTML → build-runtime.js 设为 `audioEl.src` → 浏览器解析为 `http://localhost:PORT/Users/xxx/clip.mp4` → recorder 的 resolve_media_src 去掉 base_url 前缀得到 `Users/xxx/clip.mp4` → 拼接 root → `{root}/Users/xxx/clip.mp4` → 文件不存在 → anullsrc 静音

**修复方案**:
1. `resolve_media_src` (record/mod.rs): HTTP 模式下，去掉 base_url 后检查 `/{remainder}` 是否是有效绝对路径
2. `build-runtime.js`: 设置 `window.__audioSrc` 供 recorder 优先读取（保留原始路径）
3. `mux_audio_track` (ffmpeg.rs): 加 warn log 当 fallback 到 anullsrc 时
