vox · 字级对齐对比

同一句中文,Edge TTS + 豆包(火山引擎 TTS) · 旧路径(mlx-whisper 转写)vs 新路径(whisperX 强制对齐)

原文
旧路径 mlx-whisper 对合成音频做 ASR,然后把听到的文本强塞回原文。 Whisper 会听错(例如把"傅里叶"听成"负理液"、"复理液"), 且把错的时间戳一起留下,字幕文字和时间都可能漂。
新路径 whisperX wav2vec2 CTC 强制对齐。不做 ASR,直接用原文 + 音频算每个字的声学位置。 原文 100% 保留,时间戳来自真波形。
操作 点每张卡片的播放。高亮的字 = 当前正在播放的字。对比"现在 highlight 的字对不对"和"原文是否完整"。

核心发现

旧路径在 Edge 音频上把 傅里叶 转写成 负理液——3 个字全错。 在豆包音频上把 傅里叶 转写成 复理液——同样 3 个字全错,错法还不一样。 标点(逗号、句号)要么丢失、要么被英文符号替掉。 新路径两个音频都产出 傅里叶变换和大语言模型的幻觉问题。——**一字不差,标点齐全,时间戳精准到字**。
