# Devlog: 2026-03-19 Initial State

## Snapshot

`vox` is a Rust CLI TTS tool with a stable command surface in [src/cli/mod.rs](/Users/Zhuanz/boom/vox/src/cli/mod.rs) and a backend abstraction in [src/backend/mod.rs](/Users/Zhuanz/boom/vox/src/backend/mod.rs). The code is already organized for multiple backends, but the factory currently constructs only the `edge` backend.

## Module Status

- Entry points: [src/main.rs](/Users/Zhuanz/boom/vox/src/main.rs) starts Tokio and delegates to [src/cli/mod.rs](/Users/Zhuanz/boom/vox/src/cli/mod.rs).
- Backend layer: [src/backend/mod.rs](/Users/Zhuanz/boom/vox/src/backend/mod.rs) defines `Backend`, `SynthParams`, `SynthResult`, `Voice`, and `WordBoundary`.
- Edge transport: [src/backend/edge/ws.rs](/Users/Zhuanz/boom/vox/src/backend/edge/ws.rs), [src/backend/edge/ssml.rs](/Users/Zhuanz/boom/vox/src/backend/edge/ssml.rs), and [src/backend/edge/drm.rs](/Users/Zhuanz/boom/vox/src/backend/edge/drm.rs) implement voice listing, websocket synthesis, SSML building, and token generation.
- Single-shot CLI: [src/cli/synth.rs](/Users/Zhuanz/boom/vox/src/cli/synth.rs) handles inline text, file input, stdin input, cache reuse, output writing, and optional SRT generation.
- Batch CLI: [src/cli/batch.rs](/Users/Zhuanz/boom/vox/src/cli/batch.rs) parses JSON jobs, resolves defaults, initializes backend pools, writes a manifest, and fails the process when any job fails.
- Batch scheduler: [src/queue/job.rs](/Users/Zhuanz/boom/vox/src/queue/job.rs) and [src/queue/scheduler.rs](/Users/Zhuanz/boom/vox/src/queue/scheduler.rs) own per-job backend routing, concurrency limits, cache short-circuiting, and event emission.
- Output layer: [src/output/event.rs](/Users/Zhuanz/boom/vox/src/output/event.rs), [src/output/manifest.rs](/Users/Zhuanz/boom/vox/src/output/manifest.rs), [src/output/naming.rs](/Users/Zhuanz/boom/vox/src/output/naming.rs), and [src/output/srt.rs](/Users/Zhuanz/boom/vox/src/output/srt.rs) provide NDJSON events, `manifest.json`, deterministic file names, and subtitles.
- Support modules: [src/cache/mod.rs](/Users/Zhuanz/boom/vox/src/cache/mod.rs), [src/config.rs](/Users/Zhuanz/boom/vox/src/config.rs), and [src/lang.rs](/Users/Zhuanz/boom/vox/src/lang.rs) cover caching, config persistence, and lightweight voice auto-detection.

## Backend Status

- Edge: implemented end to end in [src/backend/edge/mod.rs](/Users/Zhuanz/boom/vox/src/backend/edge/mod.rs) and constructed by [src/backend/mod.rs](/Users/Zhuanz/boom/vox/src/backend/mod.rs).
- OpenAI: not implemented in the current source tree. There is no `src/backend/openai/` module yet, although the shared `Backend` trait and CLI/backend selection flags leave room for a future OpenAI backend.

## Test Inventory

`cargo test` currently exercises 10 unit tests:

- 2 tests in [src/backend/edge/ssml.rs](/Users/Zhuanz/boom/vox/src/backend/edge/ssml.rs) for single-escape SSML handling and escaped-length chunking.
- 2 tests in [src/backend/edge/drm.rs](/Users/Zhuanz/boom/vox/src/backend/edge/drm.rs) for token and MUID formatting.
- 4 tests in [src/cli/batch.rs](/Users/Zhuanz/boom/vox/src/cli/batch.rs) for config/default resolution and batch failure propagation.
- 2 async tests in [src/queue/scheduler.rs](/Users/Zhuanz/boom/vox/src/queue/scheduler.rs) for per-job backend routing and manifest failure recording.

## Known Problems

- Multi-backend support is architectural, not complete product surface yet: [src/backend/mod.rs](/Users/Zhuanz/boom/vox/src/backend/mod.rs) rejects any backend name other than `edge`.
- `preview` is Edge-only and bypasses config/backend selection by directly constructing `EdgeBackend` in [src/cli/preview.rs](/Users/Zhuanz/boom/vox/src/cli/preview.rs).
- `concat` is a raw MP3 byte append in [src/cli/concat.rs](/Users/Zhuanz/boom/vox/src/cli/concat.rs), not a true audio remux or re-encode step.
- Live synthesis verification is still manual and network-dependent because the automated test suite covers unit behavior only; the real service path lives in [src/backend/edge/ws.rs](/Users/Zhuanz/boom/vox/src/backend/edge/ws.rs).
- Subtitle generation is conditional: [src/cli/synth.rs](/Users/Zhuanz/boom/vox/src/cli/synth.rs) writes SRT only when `--srt` is set and the backend returns word boundaries.
