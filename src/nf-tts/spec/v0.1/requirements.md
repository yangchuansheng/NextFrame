# v0.1 Requirements

## Scope

Version `0.1` is the first usable CLI release for local and agent-driven text-to-speech work. The current code already covers the core flow through [src/cli/](/Users/Zhuanz/boom/vox/src/cli) and [src/backend/](/Users/Zhuanz/boom/vox/src/backend), with `edge` as the shipped backend and room for later backends through the shared trait.

## Priority

### P0 Must Ship

- Single-sentence synthesis from inline text, files, or stdin through [src/cli/synth.rs](/Users/Zhuanz/boom/vox/src/cli/synth.rs).
- Batch synthesis from a JSON job array through [src/cli/batch.rs](/Users/Zhuanz/boom/vox/src/cli/batch.rs).
- Edge backend support for voice listing and synthesis through [src/backend/edge/mod.rs](/Users/Zhuanz/boom/vox/src/backend/edge/mod.rs) and [src/backend/edge/ws.rs](/Users/Zhuanz/boom/vox/src/backend/edge/ws.rs).
- Config management for default voice, default directory, default backend, and aliases through [src/config.rs](/Users/Zhuanz/boom/vox/src/config.rs) and [src/cli/config_cmd.rs](/Users/Zhuanz/boom/vox/src/cli/config_cmd.rs).
- Deterministic MP3 output naming and cache reuse through [src/output/naming.rs](/Users/Zhuanz/boom/vox/src/output/naming.rs) and [src/cache/mod.rs](/Users/Zhuanz/boom/vox/src/cache/mod.rs).
- Machine-readable command output: NDJSON job events and batch `manifest.json` through [src/output/event.rs](/Users/Zhuanz/boom/vox/src/output/event.rs) and [src/output/manifest.rs](/Users/Zhuanz/boom/vox/src/output/manifest.rs).
- Optional subtitle output in SRT format when timing metadata exists, via [src/output/srt.rs](/Users/Zhuanz/boom/vox/src/output/srt.rs).

### P1 Important Next

- True multi-backend delivery beyond Edge. The interface exists in [src/backend/mod.rs](/Users/Zhuanz/boom/vox/src/backend/mod.rs), but a second backend such as OpenAI is not yet implemented in `src/backend/`.
- Backend parity for preview/play behavior so [src/cli/preview.rs](/Users/Zhuanz/boom/vox/src/cli/preview.rs) no longer special-cases Edge.
- Stronger end-to-end smoke coverage for live synthesis, not only unit tests.

## Functional Requirements

### Single Synthesis

- Accept text from CLI argument, `--file`, or stdin.
- Resolve voice by explicit flag, alias, config default, or auto-detection.
- Resolve backend by explicit flag or config default.
- Write one MP3 file to the requested output directory.

### Batch Synthesis

- Accept a JSON array of jobs matching [src/queue/job.rs](/Users/Zhuanz/boom/vox/src/queue/job.rs).
- Support per-job `voice`, `rate`, `volume`, `pitch`, `backend`, and `filename`.
- Use sequential output names when `filename` is omitted.
- Write `manifest.json` and exit non-zero if any job fails.

### Multi-Backend Contract

- Keep backend-specific code behind the `Backend` trait.
- `edge` is the required backend for v0.1.
- `openai` is a target extension, not a shipped backend in the current tree.

### Output Formats

- Audio output format: MP3.
- Event output format: JSON for single synthesis, NDJSON plus final JSON summary for batch.
- Subtitle output format: SRT when word boundaries are available.
- Config file format: TOML at the path generated by [src/config.rs](/Users/Zhuanz/boom/vox/src/config.rs).
